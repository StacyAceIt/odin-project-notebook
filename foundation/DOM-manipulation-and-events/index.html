What is DOM?

DOM is a tree like tree-like representation of the contents of a webpage.

How do you target the nodes you want to work with?

Use selectors

How do events and listeners work?

  1. Events are actions that occur to your webpage. Examples:
    • mouse clicks
    • key presses
  2. Using JavaScript, we can make our webpage listen to and react to these events.

What are 3 ways to use events in your code?

  1. specify

    function attributes

    directly on HTML elements. < button onclick = "alert('Hi')" >
  2. set

    properties

    on DOM nodes. e.g. btn.onclick = () => alert('Hi')
  3. attach

    event listeners

    to DOM nodes. e.g. btn.addEventListeners("actionAsString", function);

Why are event listeners the preferred way to handle events?

  1. Separation of concerns: Using event listeners keeps your HTML clean and free from JavaScript code, which makes it easier to read and maintain.
  2. Maintainability and Modularity.
  3. You can attach multiple event handlers to a single event without overwriting existing ones. This is not possible when using inline event handlers.
  4. Event listeners can be removed easily using the removeEventListener method, providing better control over event handling.
  5. Event listeners provide more control over event handling, such as specifying whether to use the capturing or bubbling phase.
  6. Event listeners can be reused across multiple elements or dynamically added and removed as needed.

What's the differents between parentTag > childTag and parentTag childTag

  • parentTag > childTag selects the immediate child
  • parentTag childTag selects all descendants.

What is event handler (aka event listener)?

Event handler is a function that runs when event occurs.

What are 2 event flow models?

  1. Event bubbling: flow upward from the most specific element to the least specific element.
  2. Event capturing: flow downward from the least specific to the most specific.

What does event.preventDefault() do?

preventDefault is useful for stopping the default behavior of elements (like form submission, link navigation, etc.) when you want to handle the event with custom logic instead.

What does event.stopPropagation() do?

stopPropagation is useful when you want to prevent an event from reaching other event listeners that are higher up in the DOM hierarchy.

only prevents the event from propagating further in bubbling phase

What events occur in sequence when you open a webpage?

  1. DOMContentLoaded:
    1. HTML is fully loaded.
    2. DOM tree is built.
    3. external recourses like stylesheets and images haven't been loaded.
  2. load
    1. HTML and external resources are both fully loaded.

What events fire in sequence when you leave a page?

  1. beforeunload
    1. Beforeunload fires before the page and recourses are unloaded
    2. Beforeunload can be used to show confirmation dialog to confirm if the user wants to leave the page.
    3. How does beforeunload prevent data loss?
      1. user accidentally clicks a link while filling out a form.
  2. unload
    1. Unload fires when the page has completely unloaded.
    2. Unload can be used to:
      1. Send analytic data
      2. Clean up resources

    • What are some mouse events?

    1. mousedown, mouseup and click events
    2. dblclick event:
      • mousedown, mouseup and click
      • mousedown, mouseup and click
      • dblclick
    3. mousemove
      • mousemove event fires repeatedly whenever you move mouse cursor around an element.
      • mousemove event fires many times per second as the mouse is moved around, even if it's just by 1px.
      • What problem does mousemove cause?
        • mousemove may lead to performance issue if the event handler function is complex.
        • Solution:
          • add element.onmousemove = mouseMoveEventHandler; when needed
          • remove mouseMoveEventHandler when no longer needed element.onmousemove = null;
    4. mouseover & mouseout

      • Mouseover fires when the mouse cursor is outside of the element or any of its descendants and then moves inside the boundaries of the element.
      • Mouseout fires when the mouse cursor is over an element or any of its descendants and then moves to another element.
      • Mouse over and out bubble up.

    5. mouseenter & mouseleave

      • Mouse enter and leave fire up with the cursor enter or leave the element (not the descendants)
      • Mouse enter and leave do not fire up.

    6. Mnemonics

      • Over and Out Overflow: Remember "over" and "out" like overflowing water, they spill over to parents (bubble up). Enter and leave are contained within the element itself.

    7. What does it mean to register a mouse events?
      • It means to tell the browswer to keep an eye for a particular type of interaction and to execute a specific action (callback) when that interaction occurs.

    What mouse buttons do 0, 1, 2, 3, 4 represent respectively?

    1. 0: main mouse button - usually the left button
    2. 1: auxiliary button - usually middle / wheel button
    3. 2: secondary button - usually the right button
    4. 3: fourth button - usually the browser back button
    5. 4: fifth button - usually the browser forward button
