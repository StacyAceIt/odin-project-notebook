What is DOM?

DOM is a tree like tree-like representation of the contents of a webpage.

How do you target the nodes you want to work with?

Use selectors

How do events and listeners work?

  1. Events are actions that occur to your webpage. Examples:
    • mouse clicks
    • key presses
  2. Using JavaScript, we can make our webpage listen to and react to these events.

What are 3 ways to use events in your code?

  1. specify

    function attributes

    directly on HTML elements. < button onclick = "alert('Hi')" >
  2. set

    properties

    on DOM nodes. e.g. btn.onclick = () => alert('Hi')
  3. attach

    event listeners

    to DOM nodes. e.g. btn.addEventListeners("actionAsString", function);

Why are event listeners the preferred way to handle events?

  1. Separation of concerns: Using event listeners keeps your HTML clean and free from JavaScript code, which makes it easier to read and maintain.
  2. Maintainability and Modularity.
  3. You can attach multiple event handlers to a single event without overwriting existing ones. This is not possible when using inline event handlers.
  4. Event listeners can be removed easily using the removeEventListener method, providing better control over event handling.
  5. Event listeners provide more control over event handling, such as specifying whether to use the capturing or bubbling phase.
  6. Event listeners can be reused across multiple elements or dynamically added and removed as needed.
