  • What are branches in git?

    • In Git, branches are pointers to snapshots of your changes.
    • They allow you to work on different parts of a project simultaneously. Each branch can be considered an independent line of development.
    • The master or main branch is typically the default, representing the official project history.
    • Creating a new branch allows you to develop features, fix bugs, or experiment without affecting the main codebase
    • When a branch is ready, you can merge it back into the main branch. This workflow helps manage multiple development efforts efficiently.

  • How to create a new branch?

    • git branch < new-branch-name >
    • to create a new branch and switch to it git checkout -b < new-branch-name >

  • How to view all of your current branches?

    • git branch (the one with *asterisk is the current branch)
    • git branch -a (will show details)

  • How to switch to another branch?

    • git checkout < branch-name >

  • How to delete a branch?

    • if the branch has already been merged into maingit branch -d < branch-name >

    if the branch hasn't been merged into maingit branch -D < branch-name >

  • What are use cases for branches?

    1. Feature Development: Developers can create branches to work on new features independently without affecting the main codebase. This isolation helps manage incomplete or experimental features.
    2. Bug fixes: Branches allow for the quick creation of isolated environments to fix bugs. Once fixed, the changes can be merged back into the main branch.
    3. Release Management: Different branches can be used to manage various release versions, such as release-v1.0 or release-v2.0, helping in maintaining and patching old versions while developing new ones.
    4. Experimentation: Developers can experiment with new ideas or refactor code in branches without risk to the main project. If the experiment is successful, it can be merged; otherwise, the branch can be discarded.
    5. Continuous Integration/Continuous Deployment (CI/CD): Branches are integral to CI/CD workflows, allowing for automated testing and deployment of branches before merging into the main branch.
    6. Collaboration: Branches facilitate collaboration among multiple developers, enabling parallel development and easy integration of changes from different team members.

  • How to delete branch from remote repo?

    • git push origin --delete < branch-name >
