growing-and-shrinking

What are the 3 values defined in the shorthand flex property? (e.g. flex: 1 1 auto)

  1. flex-grow: the one with flex-grow value n takes n times more space than others.
  2. flex-shrink: if there is enough space, each element will not shrink. But when space is not enough, the one with greater flex-shrink value shrinks more.
  3. flex-grow and flex-shrink don't respect preset width.
  4. flex-basis

What is flex: auto;?

  • flex: auto is equivalent to flex: 1 1 auto
  • flex: auto (flex: 1 1 auto) respects the content size or specified dimensions before distributing remaining space.

What does div {flex : 1 ; } stand for?

  • flex: 1 1 0
  • flex: 1; makes items grow or shrink equally to fill all of the available space.
  • flex: 1 (flex: 1 1 0) ensures equal distribution of space without considering the content size.

What are the 3 implicit values for display: flex in the container?

  • for each child element:
  • flex: 0 1 auto;
  • flex-grow: 0 means the element will not grow. 1 means all divs will grow evenly. flex-grow: 0: This means that flex items will not grow to fill the available space in the flex container by default. They will only take up as much space as their content or specified width/height allows.
  • flex-shrink: 0 keeps divs from drinking. 1 means that all element will shrink evenly. flex-shrink: 1: This means that flex items are allowed to shrink if the flex container is not large enough to accommodate all the items. The items will shrink proportionally based on their flex-shrink values.
  • flex-basis: sets the initial size of a flex item.

What happens if align-items: center; and flex-grow: 2; are applied to the same element?

  • flex-grow: 2; will be ignored when align-items: center; is applied. Avoid applying these two to the same element.

    • What is the default value of flex-basis?

    • The default value of flex-basis is auto.
    • When the flex-basis property is set to auto, the size of the flex item is based on its content, width/height properties, or the intrinsic size of the element, whichever is defined.
    • This means the flex item will take up space according to its content or any explicit width/height values, and then the remaining space will be distributed according to the flex-grow and flex-shrink properties.

    What are the differences between flex-basis: 0%; vs flex:basis: auto;?

      flex-basis: 0%
    • Initial Size is Zero: The flex item will start with a size of zero before any additional space distribution takes place.
    • Space Distribution: The item will rely entirely on the flex-grow property to determine its size relative to other flex items. Since its initial size is zero, the remaining space in the container will be distributed based on the flex-grow values.
      flex-basis: auto;
    • Initial Size Based on Content: The flex itemâ€™s initial size is based on its content or any width/height properties specified.
    • Default Behavior: If no flex-basis is specified, it defaults to auto.
    • When flex-basis is auto, the flex-shrink property adjusts the size of the flex item relative to its content size when there is not enough space in the flex container. This leads to a more intuitive and natural shrinking behavior, where larger items shrink proportionally more than smaller ones.
    • When flex-basis is set to 0, item's width will be ignored and everything would shrink evenly. Using auto tells the item to check for a width declaration.

    For flex-grow, flex-shrink and flex-basis, which values respect the original content?

    1. flex-grow: 0;
    2. flex-shrink: 0;
    3. flex-basis: auto; only flex-basis has auto. Others don't
