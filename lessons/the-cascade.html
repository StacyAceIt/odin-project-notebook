The Cascade of CSS

Specificity

  1. Specificity order: ID > class > type > universal
  2. Having a smaller quantity of a more specific type > having greater quantity of a less specific type
  3. Combinators don't add specificity.

    4. Combinators:
    1. + (Adjacent Sibling Combinator): Selects an element that is directly after another specific element at the same level in the HTML hierarchy. For example, p + span will select a span that immediately follows a p.
    2. ~ (General Sibling Combinator): Selects all elements that follow a specified element at the same level. For instance, p ~ span will select all span elements that follow a p, regardless of their proximity.
    3. > (Child Combinator): Selects direct child elements. For example, div > p targets all p elements that are direct children of a div.

Inheritance

  1. Typography-based properties (color, font-size, font-family) are usually inherited. Most other properties aren't.
  2. A rule directly targeting a child element is the exception of inheritance.

Rule Order

  1. Rules defined at the end are the winner.
