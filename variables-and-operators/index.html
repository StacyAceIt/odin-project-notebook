8 Data Types in JS

  1. Number

  3. String
    • Strings declared using single quotes and strings declared using double quotes are the same
    • Strings declared using backticks are a special kind of string called a template literal. In most ways, template literals are like normal strings, but they have some special properties:
      • You can embed JS in them
      • you can declare template literals over multiple lines
    • Backticks are “extended functionality” quotes. They allow us to embed variables and expressions into a string by wrapping them in ${…}
      • let name = "John"; // embed a variable alert( `Hello, ${name}!` ); // Hello, John! // embed an expression alert( `the result is ${1 + 2}` ); // the result is 3
    • str.substr() vs str.slice() vs str.substr
      • slice(startIndex, endIndex): Extracts from startIndex to endIndex (not including endIndex). Accepts negative indices.
      • substr(startIndex, length): Extracts length characters from startIndex. Accepts negative startIndex.
      • substring(startIndex, endIndex): Extracts from startIndex to endIndex (not including endIndex). Swaps indices if startIndex > endIndex. Treats negative indices as 0.

Regular Expressions

  1. Creating a regular expression
    1. Regular expression literal
        2. const re = /ab+c/
      • regex literals improve performance
      • use regex literals if the regex remains constant
    2. Calling the constructor function of the RegExp object
        3. const re = new RegExp（"ab+c"）
      • Using constructor function provides runtime compilation of the regex.
      • Use the constructor function when the regex will be changing, esp. getting regex from user input.
  2. Writing a regular expression pattern
    1. Using simple patterns: /abc/ to grasp abc pattern
    2. Using special characters /a*/ 0 or more a
    3. Escaping
      • Escape: backward slash /a\*/
        • both /a\*b/ and new RegExp("a\\*b") define the same pattern. They both match the character a, followed by a literal *, followed by the character b.
        • When using the RegExp constructor in JavaScript, you need to escape the escape character because the constructor takes a string literal as its argument. In JavaScript string literals, the backslash \ is an escape character, so you must escape it with another backslash to ensure it is treated as a literal backslash in the regular expression.
      • Regex literal: forward slash /
      • Constructor function: double quotes "ab+c"

When to not use Regex?

  1. When there are parsers.
  2. More generally, when you have better tools to do your job.
  3. When parsing human writing.
  4. don't validate an e-mail address through a regular expression. In most cases, you'll do it wrong. In a rare case, you'll do it right
  5. When you don't have the right tools or don't care about clean code.
  6. When your code will be read. And then read again, and again and again, every time by different developers.
